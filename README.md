## 🚀 核心特點

- **模組化設計**：學習如何抽象化硬體，即使更換馬達，高階控制程式碼也無需修改。
- **分散式開發**：利用 TCP/IP 特性，在不同裝置（如樹莓派與 PC）之間分配運算負載。
- **實戰導向**：提供實用的程式碼片段與架構圖，幫助您快速從模擬跨入實體機器人開發。

## 🤝 如何參與協作

本指南是一個開源專案，歡迎任何有興趣的開發者共同參與完善！我們採用典型的 **Fork & Pull Request** 協作流程：

1. **Fork 本儲存庫**：您需要先擁有一個 GitHub 帳號。點擊頁面右上角的 "Fork" 按鈕，將專案複製到您自己的 GitHub 帳號下。
2. **複製到本地**：將您的 Fork 版本複製（Clone）到本地電腦進行修改。
3. **建立新分支**：為您的修改建立一個獨立的特徵分支（例如 `git checkout -b feature/your-topic`）。
4. **提交修改**：撰寫或修改內容後，提交變更並推送至您的 GitHub 儲存庫。
5. **發送 Pull Request**：在 GitHub 頁面上向本儲存庫提交 Pull Request，並簡述您的修改內容，我們會在審查後將其合併。

此外，如果您想直接加入本專案成為核心協作者（Collaborator），亦可直接聯絡 [weilingwu772](https://github.com/weilingwu772) 將您加入。

無論是修正錯字、調整格式、補充實戰經驗，還是新增章節，都非常歡迎您的貢獻！
