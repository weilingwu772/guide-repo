# 前言 (Preface)

近年來，智慧機器人技術快速發展，應用正逐步由製造場域延伸至醫療、餐飲、物流及巡檢等領域，然而在服務型應用的發展過程中，我國產業面臨到零組件供應端與整機需求端之間，仍缺乏一致性的開發與整合共識之問題；多數廠商依循自身技術路線，採用不同的通訊協定、資料格式與驅動套件，導致系統整合、功能擴充與跨廠商協作的成本居高不下，限制了台灣機器人產業生態系的形成。

在此背景下，**ROS 2 (Robot Operating System 2)** 因為其高支援特性，逐漸成為服務型機器人（Service Robot）與自主移動機器人（AMR）領域的主流開發框架，加上有鑑於感知零組件是機器人理解環境、定位導航及自主行動的重要基礎，為協助我國相關產業形成可規模化擴散的技術生態系，經濟部產業發展署與本會（財團法人資訊工業策進會）今年規劃推出聚焦於「**ROS 2 開發框架**」與「**機器人感知零組件**」兩大主軸的技術入門指引。

本指引以實務為基礎，說明依廠牌而異的感測器原始資料，如何透過 ROS 2 進行轉換、理解與融合，進而成為機器人可用的感知資訊，期望建立較一致的工程思維、開發介面與整合方法，以此做為國內機器人產業在感知技術領域的通用化基礎。後續機器人整機廠商在選用零組件時，即可快速釐清基礎需求項目，再搭配供應商提供的韌體與軟體套件（Binary），便能在開發前期以較少資源完成概念驗證，以及確認應用可行性，而不必投入大量時間處理硬體的底層整合。

雖然實務上難以做到完全「無痛開發」，但透過通用化指引與模組化設計，可以大幅降低開發門檻與導入成本，完成 POC 後，再依據實際應用需求選擇最適合量產或部署的硬體平台，工業電腦廠商提供 BSP 與 SDK，協助客戶在兼顧效能與穩定性的前提下，順利完成產品落地，並逐步接手後續的客製化與深度開發。這樣的平台定位，有助於建立開放的產業生態系，讓更多合作夥伴依循相同規範提供相容的硬體與軟體方案，共同降低客戶開發成本，縮短產品上市時程，並提升整體平台的擴充性與市場競爭力。

指引共分三章，內容綱要如下：

### **第一章 基礎 Foundation**
從實體人工智慧的未來觀點出發，盤點人型、狗型與輪型等型態機器人，在開發上對應所需的核心零組件項目及主流供應商，並描繪整體開發世界樣貌，強調模組化設計精神的重要性；接著以實務為導向，說明機器人感知系統建置的六大步驟，同時整理出可供參考的 ROS 官方開發規範 ─ REP 及既有產品慣例資訊。
